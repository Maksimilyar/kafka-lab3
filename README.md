# Лабораторна робота №3

## Розробка Kafka-продюсера для набору даних Divvy Trips

---

# Структура проєкту

```text
.
├── Divvy_Trips_2019_Q4.csv
├── docker-compose.yml
└── producer/
    ├── producer.py
    ├── requirements.txt
    └── Dockerfile
```

---

# Архітектура системи

```text
CSV-файл
    │
    ▼
Python Producer
    │
    ├────────► Topic1
    │
    └────────► Topic2
            │
            ▼
       Kafka Cluster
    (Broker1 + Broker2)
            │
            ▼
         Kafka UI
```

Кластер Apache Kafka складається з таких компонентів:

* ZooKeeper
* Broker1
* Broker2
* Kafka UI

У кластері створено два топіки:

* Topic1
* Topic2

Параметри топіків:

* Кількість партицій: 3
* Фактор реплікації: 2

---

# Опис роботи продюсера

Продюсер зчитує записи з файлу:

```text
Divvy_Trips_2019_Q4.csv
```

Кожен рядок файлу описує одну поїздку сервісу прокату велосипедів Divvy.

Для кожного запису продюсер виконує такі дії:

1. Зчитує рядок із CSV-файлу.
2. Перетворює його у JSON-об’єкт.
3. Використовує поле `trip_id` як ключ повідомлення.
4. Відправляє повідомлення у Topic1.
5. Відправляє те саме повідомлення у Topic2.

Приклад повідомлення:

```json
{
  "trip_id": 25223640,
  "start_time": "2019-10-01 00:01:46",
  "end_time": "2019-10-01 00:17:16",
  "bikeid": 2167,
  "tripduration": 930.0,
  "from_station_name": "Wabash Ave & Grand Ave",
  "to_station_name": "Milwaukee Ave & Grand Ave"
}
```

---

# Конфігурація Kafka-кластера

| Компонент | Порт |
| --------- | ---- |
| ZooKeeper | 2181 |
| Broker1   | 9092 |
| Broker2   | 9093 |
| Kafka UI  | 8080 |

Параметри топіків:

| Топік  | Партиції | Фактор реплікації |
| ------ | -------- | ----------------- |
| Topic1 | 3        | 2                 |
| Topic2 | 3        | 2                 |

---

# Запуск проєкту

Для побудови образів та запуску всіх сервісів необхідно виконати команду:

```bash
docker compose up --build
```

Після запуску веб-інтерфейс Kafka UI буде доступний за адресою:

```text
http://localhost:8080
```

Для зупинки всіх контейнерів:

```bash
docker compose down
```

# Отримані результати

У результаті виконання роботи було реалізовано Kafka-продюсер, який успішно зчитує записи з набору даних Divvy Trips та публікує їх одночасно у два Kafka-топіки:

* Topic1
* Topic2

За допомогою Kafka UI підтверджено успішне надходження повідомлень до обох топіків.

Використання двох брокерів і фактора реплікації 2 забезпечує відмовостійкість системи та збереження даних у випадку недоступності одного з брокерів.
